
# Jellyfin2HQPlayer Deployment (Linux)

1. Prepare package

Option 1: Download the release package directly to the target machine
On the target machine, use wget or curl to download the release package. For example:

wget https://github.com/YTSamLee/jellyfin2hqplayer/releases/download/v1.3.0/jellyfin2hqplayer-linux-x64-1.3.0.tar.gz
wget https://github.com/YTSamLee/jellyfin2hqplayer/releases/download/v1.3.0/deploy-jellyfin2hqplayer-linux-x64.sh
# or
curl -LO https://github.com/YTSamLee/jellyfin2hqplayer/releases/download/v1.3.0/jellyfin2hqplayer-linux-x64-1.3.0.tar.gz
curl -LO https://github.com/YTSamLee/jellyfin2hqplayer/releases/download/v1.3.0/deploy-jellyfin2hqplayer-linux-x64.sh


Option 2: Manually transfer the file to the target machine
Alternatively, you can upload the release package to the target machine using SCP, FTP, or any other preferred file transfer method. Once the file is on the target machine, proceed to the next steps.

2. Deploy

Make the script executable:
chmod +x deploy-jellyfin2hqplayer-linux-x64.sh

Auto-select latest package:
sudo ./deploy-jellyfin2hqplayer-linux-x64.sh

Or specify package:
sudo ./deploy-jellyfin2hqplayer-linux-x64.sh jellyfin2hqplayer-linux-x64-*.tar.gz

3. Port
Default:
3000

Specify port during deploy:
sudo PORT=8080 ./deploy-jellyfin2hqplayer-linux-x64.sh

4. Change Port (after deploy)
sudo nano /etc/systemd/system/jellyfin2hqplayer.service

Edit:
Environment=PORT=8080

Then run:
sudo systemctl daemon-reload
sudo systemctl restart jellyfin2hqplayer

5. Access
http://<IP>:PORT

6. Firewall (if not accessible)
Ubuntu / Debian:
sudo ufw allow 3000

Fedora:
sudo firewall-cmd --add-port=3000/tcp --permanent
sudo firewall-cmd --reload
