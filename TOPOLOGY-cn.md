# Jellyfin2HQPlayer 三种播放模式
拓扑与特性

---

## 1. 文件模式（推荐）

```text
+------------------------------+
|      浏览器（Web UI）        |
+---------------+--------------+
                |
                | HTTP
                v
+-----------------------------------+
| Jellyfin2HQPlayer Web             |
+----------------+------------------+
                 |
                 | API
                 v

+------------------------------------------------------+
| 主机 A（同一台机器）                                 |
|                                                      |
|  +------------------+   +-------------------------+  |
|  |     Jellyfin     |   |        HQPlayer         |  |
|  | （媒体库/元数据）|   |    （播放引擎）         |  |
|  +---------+--------+   +------------+------------+  |
|            |                         |               |
|            | 返回路径                | 读取文件      |
|            +-----------+-------------+               |
|                        |                             |
|                        v                             |
|         +-------------------------------+            |
|         |         本地文件系统          |            |
|         +---------------+---------------+            |
|                         |                            |
+-------------------------+----------------------------+
                          |
            +-------------+-------------+
            |                           |
            v                           v

+----------------------+   +----------------------+
| NAS 挂载路径         |   | 本地磁盘路径         |
| /mnt/NASMusic/...    |   | /local/music/...     |
+----------------------+   +----------------------+
```

### 特性

- Jellyfin 与 HQPlayer 部署在同一台机器
- HQPlayer 直接读取本地音频文件
- 无流媒体传输
- 无转码
- 最短信号路径
- 原生 bit-perfect 播放
- HQPlayer 拥有完整文件控制能力

---

## 2. HTTP 文件模式（HQPlayer 推荐）

```text
+------------------------------+
|      浏览器（Web UI）        |
+---------------+--------------+
                |
                | HTTP
                v
+------------------------------+
| Jellyfin2HQPlayer Web        |
+---------------+--------------+
                |
                | API
                v

+-----------------------------------+
| 主机 A                            |
|                                   |
|  +-----------------------------+  |
|  |         Jellyfin            |  |
|  |      （媒体库 / HTTP API）  |  |
|  +-------------+---------------+  |
+----------------|------------------+
                 |
                 | HTTP 文件（类文件访问）
                 | 顺序访问
                 | 无转码
                 v

+-----------------------------------+
| 主机 B                            |
|                                   |
|  +-----------------------------+  |
|  |         HQPlayer            |  |
|  |        （播放引擎）         |  |
|  +-------------+---------------+  |
+-----------------------------------+
```

### 特性

- Jellyfin 与 HQPlayer 可部署在不同机器
- 类文件顺序访问（file-like sequential access）
- 无转码
- HQPlayer 主动拉取音频数据
- 行为类似本地文件访问
- 保留播放控制能力
- bit-perfect

---

## 3. HTTP 流模式（兼容模式）

```text
+------------------------------+
|      浏览器（Web UI）        |
+---------------+--------------+
                |
                | HTTP
                v
+------------------------------+
| Jellyfin2HQPlayer Web        |
+---------------+--------------+
                |
                | API
                v

+-----------------------------------+
| 主机 A                            |
|                                   |
|  +-----------------------------+  |
|  |         Jellyfin            |  |
|  |      （媒体库 / HTTP API）  |  |
|  +-------------+---------------+  |
+----------------|------------------+
                 |
                 | HTTP 流
                 | Range 访问（随机读取）
                 | 无转码
                 v

+-----------------------------------+
| 主机 B                            |
|                                   |
|  +-----------------------------+  |
|  |         HQPlayer            |  |
|  |        （播放引擎）         |  |
|  +-------------+---------------+  |
+-----------------------------------+
```

### 特性

- Jellyfin 与 HQPlayer 可部署在不同机器
- HTTP 流 + Range 访问（HTTP streaming with Range access）
- 支持基于 Range 的随机位置读取
- 流媒体行为
- 兼容更多播放场景
- 无转码（正确配置时）

---

## 总结

```text
文件模式        = 本地文件直接控制（最佳）
HTTP 文件模式   = 远程但“类文件访问”（推荐）
HTTP 流模式     = 流媒体兼容方案
```
